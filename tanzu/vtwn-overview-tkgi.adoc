= VMware Tanzu Kubernetes Grid 整合版 (TKGI) 概述
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
VMware Tanzu Kubernetes Grid Integrated (TKGI) Edition（以前稱為 VMware Enterprise PKS）是基於 Kubernetes 的獨立容器編排平台，具有生命週期管理、叢集健康監控、進階網路、容器註冊表等功能。  TKGI 使用由 BOSH 和 Ops Manager 組成的 TKGI 控制平面來設定和管理 Kubernetes 叢集。

TKGI 可以在本地 vSphere 或 OpenStack 環境中安裝和運行，也可以在其各自的 IaaS 產品上的任何主要公有雲中安裝和運行。此外，TKGI 與 NSX-T 和 Harbour 的集成為企業工作負載提供了更廣泛的用例。要了解有關 TKGI 及其功能的更多信息，請訪問文檔link:https://docs.vmware.com/en/VMware-Tanzu-Kubernetes-Grid-Integrated-Edition/index.html["這裡"^]。

image:vtwn-004.png["VMware Tanzu Kubernetes Grid整合版"]

TKGI 根據不同的用例和設計以多種配置安裝在各種平台上。按照指南操作link:https://docs.vmware.com/en/VMware-Tanzu-Kubernetes-Grid-Integrated-Edition/1.14/tkgi/GUID-index.html["這裡"^]安裝並設定 TKGI 及其先決條件。  TKGI 使用 Bosh VM 作為 Tanzu Kubernetes 叢集的節點，這些叢集運行不可變的配置映像，且 Bosh VM 上的任何手動變更在重新啟動後都不會保留。

重要提示：

* NetApp Trident需要特權容器存取。因此，在 TKGI 安裝期間，請確保在配置 Tanzu Kubernetes 叢集節點計畫的步驟中選取「啟用特權容器」複選框。
+
image:vtwn-005.png["TKGI 中的特權容器"]

* NetApp建議將所有生產環境部署在多個主部署中以實現容錯，並選擇工作節點的配置來滿足預期工作負載的要求。因此，建議的 TKGI 叢集計劃將由至少三個主伺服器和三個從伺服器組成，並配備至少四個 vCPU 和 12GB RAM，以應對高強度的工作負載。
